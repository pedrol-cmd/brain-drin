# User Story Generator

## Purpose
To ensure development tasks are clearly defined, value-driven, and easy for engineers to implement.

## The Format
"As a [Persona], I want [Action], so that [Value]."

## INVEST Criteria Check
- **I**ndependent: Is it self-contained?
- **N**egotiable: Is there room for discussion?
- **V**aluable: Does it deliver user value?
- **E**stimable: Can the team size it?
- **S**mall: Does it fit in a sprint?
- **T**estable: Is there a clear pass/fail?

## Components
- **Acceptance Criteria**: Exact conditions that must be met.
- **Technical Notes**: Constraints or implementation suggestions.
- **Design Reference**: Links to Figma/Mocks.
