# Knowledge Base Scaffold

## Purpose
To reduce support burden and empower users through self-service help content.

## Architecture
1. **Getting Started**: The first 5 things every user must do.
2. **Core Concepts**: Glossary and mental models.
3. **How-to Guides**: Task-based walkthroughs.
4. **Troubleshooting**: Searchable index of common error codes and fixes.
5. **API Reference**: Technical docs.

## Rules
- Information must be searchable.
- Content must be evergreen (updated with every major UI change).
- Focus on the "Goal," not just the "Button."
