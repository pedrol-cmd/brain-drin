# Code Explain

## Context

- Target: $ARGUMENTS

## Your Task

1. **Read** the target code
2. **Explain** at three levels:
   - **What** — What does this code do? (1-2 sentences)
   - **How** — How does it work? (walk through the logic)
   - **Why** — Why is it written this way? (design decisions, tradeoffs)
3. **Highlight** anything surprising, clever, or potentially problematic
4. **Context** — How does this fit into the broader system?

## Rules

- Adjust depth to complexity. Simple code gets a simple explanation.
- Don't explain language syntax. Explain business logic and design choices.
- If something looks wrong, say so — but verify before claiming it's a bug.
